# Discord Panel Guide

## Что это

Команда `/panel` открывает единую ephemeral-панель администратора в Discord. Она не заменяет старые slash-команды полностью, но закрывает основной ежедневный поток:

- просмотр состояния сервиса и очереди;
- выбор конкретного mapping;
- пауза и снятие с паузы;
- повторный запуск последнего сообщения;
- перезапуск накопившегося хвоста failed jobs;
- просмотр glossary;
- запуск массового импорта glossary;
- создание и редактирование mapping без выхода из панели.

Важно: загрузка glossary в панели сейчас делается не файлом, а через модалку с большим текстовым полем. То же самое делает и slash-команда `/glossary import`.

Панель намеренно сделана ephemeral, чтобы не засорять служебные каналы и не открывать админские действия всем участникам сервера.

## Как открыть

1. Выполните `/panel`.
2. В верхней части панели будет краткий статус сервиса: Discord, DeepL, очередь и число активных mappings.
3. Ниже находится selector mapping и быстрые кнопки.

## Основной экран панели

После выбора mapping панель показывает:

- raw и output каналы;
- языковую пару;
- режим рендера и медиа;
- активный glossary;
- состояние mapping: активен или на паузе;
- локальную очередь по mapping: pending, retry, failed;
- последнее принятое raw-сообщение.

### Кнопки основного экрана

`Обновить`

- перечитывает текущее состояние БД и очереди;
- удобно нажимать после `/resume`, ручного изменения glossary или тестовой отправки сообщения.

`Пауза` / `Resume`

- включает или снимает паузу только у выбранного mapping;
- если mapping был автоматически остановлен после ошибки конфигурации, `Resume` возвращает его в работу.

`Retranslate latest`

- удаляет ранее опубликованный перевод последнего raw-сообщения выбранного mapping;
- ставит его заново в очередь на перевод;
- удобно после фикса glossary, DeepL-конфига или правил рендера.

`Перезапуск хвоста`

- снимает mapping с паузы, если он стоял на паузе;
- переводит до 50 failed jobs обратно в `pending`;
- помечает соответствующие записи в `failed_jobs` как переотправленные;
- это основной способ расчистить накопившийся хвост после тестов и ошибок конфигурации.

`Настроить mapping`

- открывает режим создания/редактирования mapping;
- можно использовать как для нового mapping, так и для правки текущего.

`Импорт glossary`

- открывает модалку массового импорта glossary сразу для языковой пары текущего mapping;
- после отправки payload выполняется тот же sync с DeepL, что и у старой slash-команды.
- если видите ошибку на этапе проверки пары языков, это уже ошибка DeepL glossary API, а не самой панели.

`Список glossary`

- показывает текущие активные glossary-правила для выбранной пары языков.

## Режим настройки mapping

После нажатия `Настроить mapping` панель переходит в setup-режим.

### Что можно задать прямо из панели

1. `Raw канал`
2. `Output канал`
3. `Log канал` (опционально)

После выбора каналов нажмите `Сохранить детали`. Откроется модалка с полями:

1. `source_lang`
2. `target_lang`
3. `source_label`
4. `render_mode`
5. `media_mode`

### Допустимые значения

`render_mode`

- `auto`
- `embed`
- `plain`

`media_mode`

- `auto`
- `mirror`
- `link_only`

### Кнопки setup-режима

`Сохранить детали`

- сохраняет mapping;
- обновляет guild defaults по языкам;
- применяет текущий активный glossary для выбранной пары, если он существует.

`Очистить log`

- явно сбрасывает log-канал для следующего сохранения;
- полезно, если during tests был выбран временный канал и он больше не нужен.

`Назад`

- возвращает на главный экран панели без сохранения.

## Что изменилось в публикации сообщений

Бот больше не строит заголовок embed из первой строки обычного текста. Теперь:

- если у исходного сообщения есть реальный `embed_title`, он сохраняется;
- если это обычное сообщение без embed title, публикация идёт без искусственного заголовка;
- это убирает шумный сценарий, когда простое сообщение выглядело как статья с заголовком из первого предложения.

## Как безопасно перезапустить накопившиеся сообщения

Если во время тестов накопились сообщения, которые надо прогнать заново:

1. Откройте `/panel`.
2. Выберите нужный mapping.
3. Убедитесь, что DeepL уже исправлен и mapping больше не должен падать.
4. Если mapping в паузе, нажмите `Resume` или сразу `Перезапуск хвоста`.
5. Нажмите `Перезапуск хвоста`.
6. Нажмите `Обновить` через несколько секунд и проверьте, что `failed` уменьшается, а очередь обрабатывается.
7. При необходимости нажмите `Retranslate latest`, чтобы прогнать последнее сообщение заново поверх уже исправленной конфигурации.

## Когда всё ещё нужны старые slash-команды

Панель покрывает основной админский сценарий, но старые команды всё ещё полезны:

- `/status verbose:true` для более длинной диагностики;
- `/glossary preview` для точечного просмотра правила на конкретной строке;
- `/glossary add` и `/glossary remove` для быстрых единичных правок терминов;
- `/retranslate message_id:...` если нужно повторно прогнать не последнее, а конкретное сообщение.

## Короткая операторская памятка

### Обычная работа

1. Откройте `/panel`.
2. Выберите нужный mapping.
3. Если mapping активен и очередь пуста, ничего дополнительно делать не нужно.

### Бот перестал публиковать

1. Откройте `/panel`.
2. Посмотрите строку `Статус` и очередь mapping.
3. Если mapping на паузе, сначала исправьте DeepL или glossary.
4. Нажмите `Resume`.
5. Затем нажмите `Перезапуск хвоста`.

### Нужно заново прогнать накопившиеся сообщения

1. Откройте `/panel`.
2. Выберите mapping.
3. Нажмите `Перезапуск хвоста`.
4. Для самого свежего сообщения отдельно нажмите `Retranslate latest`.

### Нужно загрузить glossary

1. Откройте `/panel`.
2. Выберите mapping с нужной парой языков.
3. Нажмите `Импорт glossary`.
4. Вставьте payload в формате из [docs/glossary-policy.md](docs/glossary-policy.md).
5. Если нужен безопасный прогон без записи, используйте slash-команду `/glossary import dry_run:true`.

## Практический сценарий админа

### Первый запуск

1. Выполнить `/panel`.
2. Нажать `Настроить mapping`.
3. Выбрать raw и output каналы.
4. В модалке указать `EN`, `RU`, при необходимости `source_label`, и оставить `render_mode=auto`, `media_mode=auto`.
5. Сохранить.
6. Отправить тестовое сообщение в raw-канал.

### После ошибки DeepL или glossary

1. Исправить Railway-переменные или glossary.
2. Выполнить `/panel`.
3. Выбрать mapping.
4. Нажать `Перезапуск хвоста`.
5. Нажать `Обновить`.
6. Проверить, что новые сообщения снова публикуются.

### После серии тестов

1. Выполнить `/panel`.
2. Выбрать mapping.
3. Посмотреть `pending`, `retry`, `failed`.
4. Нажать `Перезапуск хвоста`.
5. При необходимости отдельно нажать `Retranslate latest`.

## Ограничения

- Панель хранит короткую in-memory session примерно 30 минут. Если кнопки перестали работать, просто выполните `/panel` заново.
- Панель привязана к конкретному админу и конкретному серверу.
- Для создания и редактирования mapping бот всё ещё проверяет права в raw и output каналах; если прав не хватает, сохранение не произойдёт.
- `Перезапуск хвоста` requeue делает только для failed jobs; pending и retry jobs подхватываются воркером сами.
